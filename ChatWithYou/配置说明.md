## 二、pom.xml - Maven 项目配置

### 2.1 什么是 pom.xml

`pom.xml` 是 Maven 项目的核心配置文件，Maven 是一个**项目管理和构建工具**。它的作用是告诉 Maven：
- 项目叫什么名字
- 用什么版本的 Java 编译
- 需要哪些外部库（依赖）
- 怎么打包项目

### 2.2 核心配置项详解

#### 2.2.1 项目基本信息

```xml
<groupId>liyu</groupId>
<artifactId>ChatWithYou</artifactId>
<version>1.0-SNAPSHOT</version>
<packaging>war</packaging>
```

| 配置项 | 含义 | 说明 |
|--------|------|------|
| `groupId` | 项目组标识 | 通常是公司域名倒写，如 `com.example` |
| `artifactId` | 项目标识 | 项目名称，打包后的文件名 |
| `version` | 版本号 | `SNAPSHOT` 表示开发版本 |
| `packaging` | 打包方式 | `war` 表示 Web 应用，`jar` 表示普通 Java 项目 |

**为什么用 `war`**：Web 项目需要打成 `.war` 文件部署到 Tomcat，普通 Java 项目打成 `.jar` 文件。

#### 2.2.2 编译配置

```xml
<properties>
    <maven.compiler.source>17</maven.compiler.source>
    <maven.compiler.target>17</maven.compiler.target>
    <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
</properties>
```

| 配置项 | 含义 | 说明 |
|--------|------|------|
| `maven.compiler.source` | 源代码 Java 版本 | 你的代码用 Java 17 编写 |
| `maven.compiler.target` | 编译后字节码版本 | 生成的 class 文件兼容 Java 17 |
| `project.build.sourceEncoding` | 源码编码 | UTF-8 支持中文 |

#### 2.2.3 依赖配置（最重要）

```xml
<dependencies>
    <!-- Servlet API -->
    <dependency>
        <groupId>javax.servlet</groupId>
        <artifactId>javax.servlet-api</artifactId>
        <version>4.0.1</version>
        <scope>provided</scope>
    </dependency>
</dependencies>
```

每个依赖包含四个部分：

| 部分 | 含义 | 说明 |
|------|------|------|
| `groupId` | 库的组织名称 | 如 `javax.servlet`、`mysql` |
| `artifactId` | 库的名称 | 如 `javax.servlet-api`、`mysql-connector-java` |
| `version` | 库的版本 | 如 `4.0.1`、`8.0.33` |
| `scope` | 依赖作用域 | 控制依赖在哪些阶段生效 |

**scope 的值**：

| 值 | 含义 | 适用场景 |
|----|------|----------|
| `compile`（默认） | 编译、测试、运行都需要 | 业务代码用到的库，如 MySQL 驱动 |
| `provided` | 编译时需要，运行时由容器提供 | Servlet API、JSP API（Tomcat 自带） |
| `test` | 只在测试时需要 | JUnit 测试框架 |

**项目中的依赖及其作用**：

| 依赖 | groupId:artifactId | 作用 | scope |
|------|---------------------|------|-------|
| Servlet API | javax.servlet:javax.servlet-api | 提供 Servlet、HttpServletRequest 等类 | provided |
| JSP API | javax.servlet.jsp:javax.servlet.jsp-api | 提供 JSP 相关类 | provided |
| JSTL | javax.servlet:jstl | JSP 标准标签库，用于页面循环、条件判断 | compile |
| MySQL 驱动 | mysql:mysql-connector-java | 连接 MySQL 数据库 | compile |
| BCrypt | org.mindrot:jbcrypt | 密码哈希加密 | compile |
| Gson | com.google.code.gson:gson | JSON 序列化/反序列化 | compile |
| WebSocket API | javax.websocket:javax.websocket-api | WebSocket 实时通信 | provided |

**什么时候需要添加新依赖**：

当你写代码时遇到 `找不到符号` 或 `import 无法解析` 的错误，就需要检查是否缺少依赖：

```
// 错误示例：找不到 javax.websocket.Session
Session session;  // 需要添加 javax.websocket-api 依赖

// 错误示例：找不到 com.google.gson.Gson
Gson gson = new Gson();  // 需要添加 gson 依赖
```

#### 2.2.4 构建插件配置

```xml
<build>
    <finalName>ChatWithYou</finalName>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.11.0</version>
            <configuration>
                <source>17</source>
                <target>17</target>
                <encoding>UTF-8</encoding>
            </configuration>
        </plugin>
        
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-war-plugin</artifactId>
            <version>3.3.2</version>
            <configuration>
                <warSourceDirectory>web</warSourceDirectory>
            </configuration>
        </plugin>
    </plugins>
</build>
```

| 插件 | 作用 | 配置说明 |
|------|------|----------|
| `maven-compiler-plugin` | Java 编译插件 | 指定编译版本和编码 |
| `maven-war-plugin` | WAR 打包插件 | 指定 Web 资源目录（默认是 `src/main/webapp`，这里改成 `web`） |

**为什么需要 `<warSourceDirectory>web</warSourceDirectory>`**：

默认 Maven 项目的 Web 资源目录是 `src/main/webapp`，但我们项目的目录结构是 `web` 与 `src` 同级，所以需要告诉 Maven Web 资源在哪里。

## 三、web.xml - Web 应用配置

### 3.1 什么是 web.xml

`web.xml` 是 Java Web 应用的**部署描述符**，它告诉 Servlet 容器（如 Tomcat）：
- 有哪些 Servlet
- 每个 Servlet 对应什么 URL 路径
- 错误发生时跳转到哪里

### 3.2 核心配置项详解

#### 3.2.1 Servlet 映射

```xml
<servlet>
    <servlet-name>LoginServlet</servlet-name>
    <servlet-class>liyu.control.LoginServlet</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>LoginServlet</servlet-name>
    <url-pattern>/login</url-pattern>
</servlet-mapping>
```

| 配置项 | 含义 | 说明 |
|--------|------|------|
| `<servlet-name>` | Servlet 的名字（随便起） | 用于关联 `<servlet>` 和 `<servlet-mapping>` |
| `<servlet-class>` | Servlet 的完整类路径 | 包名 + 类名，如 `liyu.control.LoginServlet` |
| `<url-pattern>` | URL 路径 | 用户访问这个路径时，请求会转发到对应的 Servlet |

**映射关系**：当用户访问 `http://localhost:8080/ChatWithYou/login` 时，Tomcat 会创建 `LoginServlet` 的实例并调用它的 `doGet` 或 `doPost` 方法。

**两种配置方式**：

1. **web.xml 配置**（传统方式）：需要手动在 web.xml 中添加映射
2. **@WebServlet 注解**（现代方式）：直接在 Servlet 类上添加注解

```java
// 方式二：@WebServlet 注解（推荐）
@WebServlet("/login")
public class LoginServlet extends HttpServlet {
    // ...
}
```

**项目中的混合方式**：

当前项目中两种方式混用了：

| 方式 | Servlet 示例 |
|------|-------------|
| @WebServlet 注解 | LoginServlet、RegisterServlet、LogoutServlet、ProfileServlet |
| web.xml 配置 | SearchFriendServlet、FriendListServlet、SendPrivateMsgServlet 等 |

**建议**：统一使用 `@WebServlet` 注解，这样更加简洁，不需要在 web.xml 中维护映射。如果需要添加新 Servlet，推荐使用注解方式。

**项目中的 Servlet 映射**：

| Servlet 名称 | URL 路径 | 功能 |
|-------------|----------|------|
| LoginServlet | `/login` | 用户登录 |
| RegisterServlet | `/register` | 用户注册 |
| LogoutServlet | `/logout` | 用户注销 |
| ProfileServlet | `/profile` | 个人资料 |
| SearchFriendServlet | `/searchFriend` | 搜索好友 |
| SendFriendRequestServlet | `/sendFriendRequest` | 发送好友申请 |
| FriendRequestListServlet | `/friendRequestList` | 获取申请列表 |
| HandleFriendRequestServlet | `/handleFriendRequest` | 处理申请 |
| FriendListServlet | `/friendList` | 获取好友列表 |
| SendPrivateMsgServlet | `/sendPrivateMsg` | 发送私聊（HTTP） |
| GetPrivateMsgServlet | `/getPrivateMsg` | 获取私聊消息 |
| GlobalExceptionHandler | `/error` | 全局异常处理 |

#### 3.2.2 错误页面配置

```xml
<error-page>
    <exception-type>java.lang.Exception</exception-type>
    <location>/error</location>
</error-page>
<error-page>
    <error-code>404</error-code>
    <location>/error</location>
</error-page>
<error-page>
    <error-code>500</error-code>
    <location>/error</location>
</error-page>
```

| 配置项 | 含义 | 说明 |
|--------|------|------|
| `<exception-type>` | 异常类型 | 当抛出该异常时，跳转到指定页面 |
| `<error-code>` | HTTP 错误码 | 当发生该错误时，跳转到指定页面 |
| `<location>` | 跳转路径 | 错误处理 Servlet 的路径 |

**作用**：当应用抛出异常或返回 HTTP 错误时，统一跳转到 `/error` 路径，由 `GlobalExceptionHandler` 处理，给用户友好的错误提示。

#### 3.2.3 什么时候需要修改 web.xml

1. **添加新 Servlet**：当你创建了一个新的 Servlet 类，并且没有使用 `@WebServlet` 注解时
2. **配置错误处理**：添加新的错误码或异常类型处理
3. **配置初始化参数**：给 Servlet 传递启动参数

## 四、db.properties - 数据库连接配置

### 4.1 什么是 db.properties

`db.properties` 是一个**键值对配置文件**，用于存储数据库连接信息。把数据库配置放在单独的文件中，而不是硬编码在代码里，有以下好处：

1. **安全性**：避免密码泄露到代码仓库
2. **灵活性**：切换环境（开发/测试/生产）时只需改配置文件
3. **可读性**：配置一目了然，方便修改

### 4.2 配置项详解

```properties
# MySQL驱动类（MySQL8.0及以上用这个）
driver=com.mysql.cj.jdbc.Driver
# 数据库连接地址，替换成你自己的数据库IP和端口
url=jdbc:mysql://localhost:3306/chat_with_you?useUnicode=true&characterEncoding=utf8&serverTimezone=GMT%2B8
# 数据库用户名
username=root
# 数据库密码，替换成你自己的MySQL密码
password=123456
```

| 配置项 | 含义 | 说明 |
|--------|------|------|
| `driver` | JDBC 驱动类名 | MySQL 8.0+ 使用 `com.mysql.cj.jdbc.Driver`，5.x 使用 `com.mysql.jdbc.Driver` |
| `url` | 数据库连接 URL | `jdbc:mysql://主机:端口/数据库名?参数` |
| `username` | 数据库用户名 | 通常是 `root` |
| `password` | 数据库密码 | 你安装 MySQL 时设置的密码 |

**URL 参数说明**：

| 参数 | 作用 |
|------|------|
| `useUnicode=true` | 使用 Unicode 字符集 |
| `characterEncoding=utf8` | 字符编码为 UTF-8 |
| `serverTimezone=GMT%2B8` | 设置时区为东八区（解决时间差问题） |

### 4.3 如何读取 db.properties

在 [JDBCUtils.java](file:///d:/java代码/mystatic/Liyu/ChatWithYou/src/main/java/liyu/util/JDBCUtils.java) 中读取配置：

```java
public class JDBCUtils {
    private static String driver;
    private static String url;
    private static String username;
    private static String password;

    static {
        try {
            Properties props = new Properties();
            // 从类路径加载 db.properties
            props.load(JDBCUtils.class.getClassLoader().getResourceAsStream("db.properties"));
            driver = props.getProperty("driver");
            url = props.getProperty("url");
            username = props.getProperty("username");
            password = props.getProperty("password");
            Class.forName(driver);
        } catch (Exception e) {
            throw new RuntimeException(e);
        }
    }

    public static Connection getConnection() {
        return DriverManager.getConnection(url, username, password);
    }
}
```

**工作流程**：
1. 类加载时（`static` 块）读取 `db.properties`
2. 解析键值对，赋值给静态变量
3. 注册 JDBC 驱动
4. 调用 `getConnection()` 时使用这些配置创建数据库连接

### 4.4 什么时候需要修改 db.properties

1. **更换数据库密码**：修改 `password` 字段
2. **更换数据库地址**：修改 `url` 中的主机和端口
3. **更换数据库名称**：修改 `url` 中的数据库名
4. **切换数据库类型**：修改 `driver` 和 `url`（如从 MySQL 切换到 PostgreSQL）

## 六、常见配置问题排查

### 6.1 编译错误：找不到符号

**问题**：`Error:(9, 17) java: 找不到符号 符号: 类 ServerEndpoint`

**原因**：缺少 WebSocket API 依赖

**解决**：在 pom.xml 中添加依赖

```xml
<dependency>
    <groupId>javax.websocket</groupId>
    <artifactId>javax.websocket-api</artifactId>
    <version>1.1</version>
    <scope>provided</scope>
</dependency>
```

### 6.2 运行错误：数据库连接失败

**问题**：`SQLException: Access denied for user 'root'@'localhost'`

**原因**：db.properties 中的用户名或密码错误

**解决**：修改 db.properties 中的 `username` 和 `password`

### 6.3 404 错误：Servlet 找不到

**问题**：访问 `/login` 返回 404

**原因**：
1. Servlet 没有在 web.xml 中配置映射
2. 使用了 `@WebServlet` 注解但路径写错
3. Servlet 类没有继承 `HttpServlet`

**解决**：检查 Servlet 配置或注解

### 6.4 Maven 依赖下载失败

**问题**：pom.xml 中依赖标红，提示无法下载

**解决**：
1. 点击 IDEA 右侧 Maven 面板的 **Reload All Maven Projects**
2. 检查网络连接
3. 清理缓存：`File → Invalidate Caches → Invalidate and Restart`

## 八、总结

| 配置文件 | 回答的问题 |
|----------|-----------|
| pom.xml | 项目用什么库？怎么编译？怎么打包？ |
| web.xml | 请求怎么路由？错误怎么处理？ |
| db.properties | 数据库在哪里？怎么连接？ |

**核心原则**：配置文件是代码和环境之间的桥梁，把变化的部分（如密码、路径、依赖版本）从代码中分离出来，让代码更加灵活和安全。
